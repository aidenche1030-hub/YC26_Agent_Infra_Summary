    核心体量 (Traction)

    DEEP DIVE

    核心任务

    应用场景

    核心体量 (Traction)
    API日均处理超 25亿 Token，实现 $80k MRR。服务 45 家头部 AI-Native 应用开发商，平均为客户节省 35% 成本。
    Sector Insight
    这个赛道的核心是帮助企业省钱。无论是通过软件层面的 Token 压缩，还是底层的显卡任务自动调度，目的都是为了在无法获取更多高端硬件的情况下，把现有的计算成本降到最低。

    02. 身份与权限：企业的授权管理

    DEEP DIVE
    Agentic Fabriq
    核心任务
    • 为企业内部的 AI Agent 提供统一的身份认证和权限管理系统。
    • 防止 Agent 拿着高级别用户的最高权限到处跑，为不同的 AI 分配特定的动态鉴权令牌。
    应用场景
    • 系统能做到全链路溯源：如果 AI 误删了数据库记录，管理员能通过日志直接查出是哪个模型、在哪个员工的授权下做的操作。
    • 没有这套管理机制，大型企业无法向审计部门解释 AI 操作的合规性（SOC2 审计）。
    核心体量 (Traction)
    日均处理 150万次 鉴权请求，达成 $40k MRR。目前在与 20 余家金融/医疗企业跑 Pilot。
    DEEP DIVE
    Didit
    核心任务
    为 C 端用户和他们的 AI Agent 提供跨平台的偏好数据和身份打通服务。
    应用场景
    • 目前每个网站的 AI 都是孤立的。用户在 A 网站的 AI 那里留下了“喜欢靠窗座位、衣服穿 M 码”的数据，去了 B 网站还要重新输入。
    • Didit 把用户的个人信息、支付方式、偏好数据打包在一起。用户授权后，任何外部平台的 Agent 都可以读取这个身份包，直接按照用户的喜好去执行订票或购物任务，省去重复交互的环节。
    核心体量 (Traction)
    开源 SDK 月下载量突破 50万 次，覆盖 15万 活跃开发者，商业版 API 实现 $15k MRR。
    Sector Insight
    企业在让 AI 代替人类操作系统前，必须解决权限分配的问题。这套身份网络明确了 AI 在系统里的操作身份，确保所有的执行动作都在被允许的范围内，并且可以随时追踪和撤销权限。

    03. 安全与护栏：API 拦截与沙盒隔离

    DEEP DIVE
    Salus
    核心任务
    提供 API 请求的前置拦截器。在 AI Agent 真正执行外部调用（如发送邮件、转账、修改数据库）之前，强制进行安全合规验证。
    应用场景
    • 传统企业安全软件只能在操作发生后记录日志并报警。
    • Salus 会在请求发出的前一刻校验参数是否合规，如果违规直接阻断，并向大模型返回具体的错误原因，让大模型纠正后重新执行。
    核心体量 (Traction)
    已累计评估 50亿+ 次高危 API 调用，营收 $65k MRR，成功打入 8 家财富 500 强企业。
    DEEP DIVE
    Runtime
    核心任务
    为自动编写代码的 AI Agent 提供云端隔离容器（沙盒），专门用来安全地运行和测试未知代码。
    应用场景
    • AI 程序员（如 Devin）生成的代码如果直接在企业的真实服务器上运行，可能会引发系统崩溃或引入恶意程序。
    • Runtime 提供用完即毁的独立云环境。AI 可以在里面自由跑代码测试，无论代码出错还是引发崩溃，都不会影响企业的主代码库和系统。
    核心体量 (Traction)
    日均自动生成/销毁超 10万 个沙盒，达成 $25k MRR，开发者排队名单超 1,500 人。
    Sector Insight
    过去企业安全主要防御外部黑客攻击，现在的重点是防止内部 AI 程序因为逻辑错误导致的破坏。如果不加拦截，AI 写错的代码会导致系统崩溃，错误的 API 调用会导致数据泄露。这一赛道专门在 AI 执行具体动作前进行验证拦截和物理隔离。

    04. 监控与可靠性：状态追踪与自动化测试

    DEEP DIVE
    Sentrial
    核心任务
    专门监控 AI Agent 在执行多步复杂任务时的逻辑状态，检测并阻断陷入死循环或偏离任务目标的行为。
    应用场景
    • 传统的服务器监控只能看到 CPU 或内存的使用率，无法判断 AI 的业务逻辑是否出错。
    • 当给 AI 安排一个 10 步的任务时，它可能在第 3 步和第 4 步之间反复循环做无用功。Sentrial 追踪每一步的推理日志，一旦发现重复动作，立刻强制停止并通知开发者。
    核心体量 (Traction)
    部署超 50个 企业级监控节点，营收 $40k MRR，正为 300+ 个高频生产级 Agent 提供监控。
    DEEP DIVE
    Moda
    核心任务
    提供自动化测试平台。使用高级大模型作为评估工具，按照开发者设定的指标，批量跑分测试 Agent 的输出质量。
    应用场景
    • 开发者修改了提示词或更换底层模型后，过去只能依靠人工随机抽查聊天记录来判断回答质量，效率很低。
    • Moda 可以根据几千条历史对话记录自动生成测试集，由系统自动打分出具报告，让开发者直观看到这次修改有没有导致 AI 退步。
    核心体量 (Traction)
    服务 60+ AI 团队，API 日调用量破 1000万，达成 $45k MRR。
    Sector Insight
    监控传统软件是看有没有报错和宕机，监控 AI 软件是看它的回答质量和任务逻辑。这部分基础设施通过抓取大模型每一步的推理过程，并利用自动化测试流水线，大幅节省了企业人工排查 AI 错误和评估回答质量的时间。

    05. 金融支付：虚拟卡与语音结算

    DEEP DIVE
    Sponge
    核心任务
    为 AI Agent 程序提供可调用 API 的虚拟信用卡，并为企业提供配套的资金额度管理后台。
    应用场景
    • 传统银行开户需要真人实名验证（KYC）。Sponge 解决了针对代码程序的合规审查问题。
    • 企业可以给负责采购的 AI 分发一张专属虚拟卡，并设定“每月最高限额 500 美元用于支付服务器费用”的规则，让 AI 在不需要人工干预的情况下完成自动续费和结算。
    核心体量 (Traction)
    平台月均处理 Agent 交易额突破 $250k，营收 $12k MRR，80+ 生产级 Agent 在使用其账户。
    DEEP DIVE
    Maven
    核心任务
    提供语音 AI 专用的支付网关接口。将声纹识别与支付系统打通，实现通话过程中的直接扣款。
    应用场景
    • 传统流程下，用户通过电话让 AI 预订服务，最后一步必须挂断电话并点击手机短信里的支付链接付款，这会导致大量用户流失。
    • Maven 接口允许用户在通话中直接说出支付确认指令，系统实时验证声纹并调取绑定的信用卡扣款，实现了不跨页面的纯语音支付闭环。
    核心体量 (Traction)
    使客户语音支付转化率提升 40%，日均处理 $50k 语音流水，与 3 家北美头部连锁酒店达成合作。
    Sector Insight
    如果 AI 要独立完成完整的业务闭环（例如订票、买资源），就必须具备付款能力。但现有的支付系统主要是基于人类用户设计的。这批公司专门提供供机器代码调用的支付 API 和虚拟卡开户服务，解决了 AI 自动化执行的最后一环。

    06. 环境与交互：后台云环境与前端组件

    DEEP DIVE
    Terminal Use
    核心任务
    为 AI Agent 提供独立运行的云端主机环境和持久化的文件系统。
    应用场景
    • 一般网页端 AI，只要用户关闭浏览器页面，任务就会中断，所有上下文都会丢失。
    • 有些任务（如大规模数据爬取、深度市场分析报告）需要耗时数小时甚至几天。Terminal Use 提供了一个后台运行环境，用户布置完任务后可以关闭网页，AI 会在云端继续执行，并把最终生成的文件保存在这个环境中。
    核心体量 (Traction)
    累计为离线 Agent 提供超 150万 小时运行环境，活跃 AI DAU 破 5,000，实现 $25k MRR。
    DEEP DIVE
    21st
    核心任务
    提供供大模型调用的前端 UI 组件代码库。让大模型可以直接在聊天界面中渲染出可视化图表或交互式按钮。
    应用场景
    • 如果用户询问财务数据或需要填写信息，大模型仅返回纯文本是不够直观的。
    • 通过接入 21st 的接口，AI 可以直接在回复中返回组件代码。用户的聊天框里会立刻出现一个可拖拽的柱状图或一个带有日历选择器的表单，避免了开发者需要自己手动编写这些前端界面的解析逻辑。
    核心体量 (Traction)
    UI 组件库月均被调用 300万次，服务 1,200+ 企业开发者，营收 $55k MRR。
    Sector Insight
    AI 要真正发挥效用，需要突破传统聊天框的限制。这部分的基础设施一方面提供了满足耗时任务所需的长期后台运行环境，另一方面提供了提升用户视觉体验的前端渲染工具库，让 AI 产品的交互从“纯文本对话”升级为“可视化应用”。

    总结：Agent Infra 的商业化实质

    从以上板块的分布可以看出：Agent 正在从聊天工具变成能独立执行业务流程的程序。

    核心演变方向 具体业务变化
    从降低成本 到 提升操作权限 早期做基础设施是为了降低大模型 API 的调用费用；现在做基础设施是为了让 AI 能安全地执行高风险操作（如修改代码库、支付转账）。
    从盲目信任 到 操作可追溯 企业无法接受 AI 的黑盒操作。现在的产品重点是记录 AI 每一步的推理日志、验证它的 API 请求，满足企业的审计和数据合规要求。
    从聊天助手 到 自动化流程节点 AI 拥有了独立的后台运行环境、虚拟信用卡账户和跨平台身份标识。这意味着 AI 程序已经可以作为独立的主体去采购资源并完成业务闭环。

    "在这个赛道中，谁能为 AI 提供最稳定的运行环境、最合规的接口拦截以及最详细的操作日志，谁就能成为下一代企业级应用的基础底座。"
